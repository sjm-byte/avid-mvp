# 04 - User Flows: جریان‌های کاربری آوید

نسخه: `v0.1-mvp`

## 1. جریان بازدیدکننده تا ثبت‌نام

```text
صفحه اصلی
  → مشاهده معرفی آوید
  → مشاهده پروژه‌های فعال/خاتمه‌یافته
  → ورود به صفحه جزئیات پروژه
  → مشاهده هشدار ریسک عمومی
  → کلیک روی «درخواست بررسی/مشارکت»
  → ثبت‌نام/ورود
  → انتقال به صفحه پروژه
```

هدف: کاربر قبل از ثبت‌نام حس اعتماد، شفافیت و جدی بودن محصول را دریافت کند.

## 2. جریان انتخاب پروژه توسط سرمایه‌گذار

```text
ورود سرمایه‌گذار
  → داشبورد
  → لیست پروژه‌ها
  → صفحه جزئیات پروژه
  → مطالعه توضیحات، ریسک‌ها، اسناد و سناریوهای سود
  → پذیرش هشدار ریسک پروژه
  → پذیرش قرارداد/شرایط پروژه
  → وارد کردن مبلغ پیشنهادی مشارکت
  → ثبت درخواست
  → وضعیت: submitted
```

نکته: سرمایه‌گذار نمی‌تواند بدون پذیرش ریسک و قرارداد، درخواست مشارکت ثبت کند.

## 3. جریان تأیید درخواست توسط ادمین

```text
ادمین وارد پنل می‌شود
  → درخواست‌های مشارکت
  → مشاهده درخواست جدید
  → بررسی پروفایل سرمایه‌گذار و مبلغ
  → تأیید یا رد
  → اگر تأیید شد: status = approved_by_admin
  → ارسال/فعال‌سازی اطلاعات واریز
  → status = payment_instructions_sent
  → ایجاد اعلان برای سرمایه‌گذار
```

## 4. جریان واریز و ثبت رسید

```text
سرمایه‌گذار اعلان دریافت می‌کند
  → صفحه درخواست مشارکت
  → مشاهده اطلاعات واریز پروژه
  → پرداخت خارج از سیستم به حساب معرفی‌شده
  → آپلود رسید یا ثبت شماره پیگیری
  → status = receipt_uploaded
  → اعلان برای ادمین
```

## 5. جریان تأیید رسید

```text
ادمین/مالی وارد بخش رسیدها می‌شود
  → مشاهده رسیدهای pending
  → بررسی مبلغ، شماره پیگیری، حساب مقصد، تاریخ
  → تأیید رسید
  → status receipt = verified
  → status request = payment_verified
  → ایجاد investment
  → ایجاد ledger_entries:
      payment_verified
      capital_allocated
  → محاسبه مجدد funding_summary پروژه
  → اعلان برای سرمایه‌گذار
```

اگر رسید رد شود:

```text
ادمین دلیل رد را وارد می‌کند
  → status receipt = rejected
  → status request = payment_instructions_sent
  → اعلان با دلیل رد برای سرمایه‌گذار
```

## 6. جریان اجرای پروژه و گزارش‌دهی

```text
ادمین وارد پروژه می‌شود
  → افزودن گزارش جدید
  → انتخاب نوع گزارش: عملیاتی/مالی/ریسک/تأخیر/تسویه
  → تعیین visibility
  → انتشار گزارش
  → ایجاد اعلان برای سرمایه‌گذاران همان پروژه
```

سرمایه‌گذار:

```text
داشبورد
  → گزارش‌های جدید
  → ورود به پروژه من
  → مشاهده Timeline، گزارش‌ها، اسناد و وضعیت سرمایه
```

## 7. جریان ثبت هزینه و درآمد پروژه

```text
ادمین/مالی
  → پروژه
  → Ledger پروژه
  → ثبت هزینه یا درآمد
  → انتخاب نوع رویداد
  → وارد کردن مبلغ، توضیح، سند پیوست
  → ثبت Ledger Entry
  → به‌روزرسانی گزارش مالی پروژه
```

## 8. جریان بستن پروژه

```text
ادمین
  → پروژه در وضعیت in_execution/delayed
  → ثبت درآمد نهایی
  → ثبت هزینه‌های نهایی
  → ثبت کارمزدهای آوید
  → محاسبه نتیجه نهایی
  → مرور لیست سهم سرمایه‌گذاران
  → تأیید نهایی
  → ایجاد settlement برای هر سرمایه‌گذار
  → تغییر وضعیت پروژه به settlement_in_progress
```

## 9. جریان محاسبه سهم سرمایه‌گذار

```text
کل سرمایه تأییدشده پروژه = sum(investments.verified_amount)
سهم هر سرمایه‌گذار = verified_amount / total_verified_capital
نتیجه قابل تقسیم = درآمد - هزینه‌ها - کارمزد موفقیت آوید
سهم سود/زیان سرمایه‌گذار = نتیجه قابل تقسیم × سهم سرمایه‌گذار
مبلغ قابل تسویه = اصل سرمایه + سهم سود/زیان
```

## 10. جریان تسویه

```text
ادمین/مالی
  → تسویه‌ها
  → مشاهده لیست settlementهای پروژه
  → ثبت پرداخت به سرمایه‌گذار
  → وارد کردن مبلغ، تاریخ، شماره پیگیری
  → status = partially_paid یا paid
  → ledger_entry = settlement_paid
  → اعلان برای سرمایه‌گذار
```

سرمایه‌گذار:

```text
داشبورد
  → تسویه‌های من
  → مشاهده مبلغ قابل دریافت، مبلغ پرداخت‌شده، شماره پیگیری
```

## 11. جریان شفافیت عمومی

```text
بازدیدکننده/سرمایه‌گذار
  → صفحه شفافیت
  → مشاهده پروژه‌های خاتمه‌یافته
  → فیلتر بر اساس نوع پروژه، سال، سود واقعی، وضعیت
  → ورود به گزارش عمومی پروژه
  → مشاهده پیش‌بینی اولیه، نتیجه واقعی، دلیل اختلاف، مدت واقعی، گزارش نهایی
```

## 12. جریان مدیریت اسناد

```text
ادمین
  → پروژه
  → مرکز اسناد
  → آپلود سند
  → انتخاب نوع سند
  → انتخاب سطح نمایش: عمومی/فقط سرمایه‌گذاران/فقط ادمین
  → ذخیره
```

## 13. جریان اصلاح داده مالی

چون حذف Ledger ممنوع است:

```text
ادمین/مالی متوجه اشتباه می‌شود
  → ایجاد entry جدید از نوع adjustment
  → توضیح دلیل اصلاح
  → اتصال به reference entry اشتباه
  → ثبت audit_log
```

## 14. جریان پیشنهادی Demo برای MVP

برای نمایش به سرمایه‌گذار یا تیم:

```text
۱. ورود به صفحه اصلی آوید
۲. مشاهده سه پروژه نمونه
۳. ورود به پروژه واردات مواد اولیه
۴. مشاهده ریسک‌ها، تدابیر، سناریوها و اسناد
۵. ثبت‌نام سرمایه‌گذار
۶. پذیرش ریسک و ثبت درخواست ۵۰ میلیون تومان
۷. ورود ادمین و تأیید درخواست
۸. آپلود رسید توسط سرمایه‌گذار
۹. تأیید رسید توسط ادمین
۱۰. مشاهده پروژه در داشبورد سرمایه‌گذار
۱۱. انتشار یک گزارش پروژه
۱۲. بستن پروژه با سود واقعی
۱۳. مشاهده سهم سود و تسویه سرمایه‌گذار
```
