# 07 - Development Plan: برنامه توسعه آوید

نسخه: `v0.1-mvp`

## 1. اصل توسعه

آوید نباید یک‌جا ساخته شود. باید Sprint به Sprint جلو رفت تا هر بخش قابل تست و اصلاح باشد. هدف MVP این نیست که تمام آرزوهای محصول را بسازد؛ هدف این است که عملیات واقعی فعلی را از حالت دستی و سنتی به یک سیستم شفاف و قابل مدیریت منتقل کند.

## 2. Stack پیشنهادی

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui
- Supabase Auth
- Supabase PostgreSQL
- Supabase Storage
- Vercel

## 3. Sprintها

### Sprint 1: اسکلت پروژه و احراز هویت

هدف: پروژه فنی بالا بیاید و ساختار اصلی آماده شود.

خروجی‌ها:

- Next.js project
- Tailwind + shadcn
- RTL setup
- Supabase client
- Auth pages
- Layout عمومی
- Layout پنل
- نقش‌های اولیه
- Dashboard خالی

معیار پذیرش:

- کاربر بتواند ثبت‌نام و ورود کند.
- ادمین و سرمایه‌گذار مسیر جداگانه داشته باشند.
- UI فارسی RTL درست نمایش داده شود.

### Sprint 2: پروژه‌ها و ویترین

هدف: نمایش پروژه‌ها و صفحه جزئیات پروژه.

خروجی‌ها:

- صفحه اصلی
- لیست پروژه‌ها
- صفحه جزئیات پروژه
- کارت پروژه
- نمایش ریسک‌ها، سناریوها، اسناد و زمان‌بندی
- Seed data برای ۳ پروژه

معیار پذیرش:

- کاربر عمومی بتواند پروژه‌ها را ببیند.
- صفحه پروژه هشدار سود تضمینی نبودن را نشان دهد.

### Sprint 3: درخواست مشارکت و پذیرش ریسک

هدف: سرمایه‌گذار بتواند یک پروژه را انتخاب کند و درخواست مشارکت ثبت کند.

خروجی‌ها:

- Modal/flow ثبت درخواست
- پذیرش ریسک
- پذیرش قرارداد
- جدول investment_requests
- داشبورد اولیه سرمایه‌گذار
- اعلان داخلی اولیه

معیار پذیرش:

- بدون پذیرش ریسک، درخواست ثبت نشود.
- درخواست در پنل ادمین دیده شود.

### Sprint 4: رسید، تأیید پرداخت و Ledger

هدف: پرداخت خارج از سیستم ثبت و کنترل شود.

خروجی‌ها:

- اطلاعات واریز پروژه
- آپلود رسید
- پنل ادمین برای بررسی رسیدها
- تأیید/رد رسید
- ایجاد investment پس از تأیید
- ایجاد Ledger entries
- محاسبه funding summary

معیار پذیرش:

- با تأیید رسید، پروژه در داشبورد سرمایه‌گذار نمایش داده شود.
- Ledger قابل مشاهده باشد.

### Sprint 5: گزارش پروژه و مرکز اسناد

هدف: سرمایه‌گذار بتواند وضعیت پروژه را دنبال کند.

خروجی‌ها:

- افزودن گزارش پروژه توسط ادمین
- نمایش گزارش‌ها برای سرمایه‌گذاران مجاز
- آپلود اسناد پروژه
- سطح دسترسی سند
- Timeline پروژه
- اعلان گزارش جدید

معیار پذیرش:

- سرمایه‌گذار فقط گزارش پروژه‌های خودش را ببیند.
- اسناد عمومی/خصوصی درست تفکیک شوند.

### Sprint 6: نتیجه نهایی، محاسبه سود/زیان و تسویه

هدف: پروژه بسته شود و سهم افراد محاسبه شود.

خروجی‌ها:

- ثبت درآمد و هزینه نهایی پروژه
- ثبت کارمزد آوید
- Preview تسویه
- Finalize settlement
- ثبت پرداخت تسویه
- صفحه پروژه‌های خاتمه‌یافته
- صفحه شفافیت

معیار پذیرش:

- سهم هر سرمایه‌گذار درست محاسبه شود.
- پروژه بسته‌شده در صفحه شفافیت نمایش داده شود.

## 4. فازهای بعد از MVP

### فاز 2: عملیاتی‌سازی جدی

- نقش مدیر مالی جداگانه
- تیکتینگ
- پیامک/ایمیل واقعی
- خروجی اکسل کامل
- گزارش‌های مالی پیشرفته
- احراز هویت رسمی
- پنل سرمایه‌پذیر

### فاز 3: حرفه‌ای‌سازی

- امضای دیجیتال
- BI و تحلیل داده
- امتیازدهی پروژه بعد از پایان
- API برای حسابداری خارجی
- اپلیکیشن موبایل
- گزارش حسابرس
- گردش‌کار حقوقی کامل

## 5. کارهای ممنوع در MVP

- درگاه پرداخت واقعی
- کیف پول واقعی
- نگهداری پول در حساب آوید
- بازار ثانویه
- رأی‌گیری سرمایه‌گذاران
- اتوماسیون مالیاتی
- وعده سود تضمینی در UI

## 6. تست‌های ضروری

### تست محصول

- سرمایه‌گذار بدون پذیرش ریسک نمی‌تواند درخواست دهد.
- درخواست ردشده به مرحله پرداخت نمی‌رود.
- رسید ردشده investment ایجاد نمی‌کند.
- تأیید رسید investment و ledger ایجاد می‌کند.
- سرمایه‌گذار اطلاعات دیگری را نمی‌بیند.
- ادمین می‌تواند پروژه را از جذب تا تسویه جلو ببرد.

### تست محاسبات

- جمع investmentها با funding summary برابر باشد.
- درصد مالکیت‌ها مجموعاً ۱۰۰٪ یا نزدیک به آن باشد.
- سود/زیان به نسبت سهم تقسیم شود.
- کارمزد موفقیت فقط در سود مثبت محاسبه شود.
- settlement_paid بیش از total_due نشود.

### تست امنیت

- Investor نتواند endpoint ادمین را صدا بزند.
- Investor نتواند رسید دیگران را ببیند.
- Investor نتواند Ledger ایجاد کند.
- Ledger posted حذف نشود.

## 7. تعریف Done برای MVP

MVP آماده استفاده محدود است وقتی:

1. سه پروژه نمونه/واقعی ثبت شده‌اند.
2. حداقل یک مسیر کامل از درخواست تا تسویه تست شده است.
3. پنل سرمایه‌گذار و ادمین بدون داده دستی خارج از سیستم قابل استفاده‌اند.
4. Ledger همه رویدادهای مالی مهم را ثبت می‌کند.
5. صفحه شفافیت برای پروژه‌های بسته‌شده کار می‌کند.
6. متن‌های هشدار ریسک در نقاط حساس نمایش داده می‌شوند.
