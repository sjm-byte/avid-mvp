# Avid Build Pack

نسخه: `v0.1-mvp`
تاریخ: ۱۴۰۵/۰۴/۱۸
زبان محصول: فارسی، راست‌به‌چپ

این پکیج برای ساخت MVP آوید آماده شده است: یک پلتفرم مدیریت مشارکت پروژه‌ای که سرمایه‌گذاران بتوانند هر پروژه را جداگانه ببینند، انتخاب کنند، قرارداد/ریسک را بپذیرند، رسید پرداخت خود را ثبت کنند و وضعیت پروژه و سهم خود را دنبال کنند.

## فرض‌های قطعی این نسخه

1. آوید فعلاً محصول حقوقی نهایی یا صندوق رسمی نیست؛ MVP برای سر و شکل دادن به عملیات فعلی و مدیریت داخلی است.
2. پول در حساب آوید نمی‌چرخد. پرداخت سرمایه‌گذار به حساب مجری پروژه یا حساب معرفی‌شده برای همان پروژه انجام می‌شود.
3. آوید با سرمایه‌گذار قرارداد دارد و در نقش وکیل/نماینده او عمل می‌کند.
4. آوید جداگانه با سرمایه‌پذیر/مجری پروژه قرارداد می‌بندد.
5. سود تضمینی نیست و پس از پایان پروژه بر اساس حساب‌وکتاب واقعی تقسیم می‌شود.
6. سرمایه‌گذار در MVP فقط پروژه را انتخاب می‌کند؛ رأی‌گیری و مداخله در تصمیمات اجرایی فعلاً وجود ندارد.
7. ریسک‌ها کیفی توضیح داده می‌شوند، نه با امتیاز عددی.
8. درآمد آوید از دو بخش است: کارمزد اولیه از سرمایه‌پذیر و درصدی از سود محقق‌شده در پایان پروژه.

## ترتیب پیشنهادی استفاده

برای ساخت با Cursor:

1. ابتدا `README.md` و `.cursor/rules/avid-product-rules.mdc` را داخل پروژه قرار بده.
2. سپس فایل‌های `docs/01` تا `docs/07` را به Cursor بده.
3. اجرای توسعه را فقط Sprint به Sprint شروع کن.
4. از `docs/08-cursor-prompts/sprint-01.md` شروع کن.
5. بعد از هر Sprint خروجی را Review کن و بعد برو Sprint بعدی.

برای ساخت با Lovable:

1. فایل `docs/09-lovable-master-prompt.md` را کامل به Lovable بده.
2. بعد از ساخت اولیه، از فایل‌های PRD و UI Spec برای اصلاح صفحه‌ها استفاده کن.
3. پرداخت واقعی، کیف پول واقعی و اتصالات بانکی را در MVP فعال نکن.

## نکته حقوقی

این پکیج سند فنی/محصولی است و جایگزین مشاوره حقوقی، مالیاتی، بازار سرمایه یا قراردادی نیست. چون پول واقعی و عموم مردم درگیر هستند، قبل از عرضه عمومی باید متن قراردادها، هشدار ریسک، مدل کارمزد، مسیر پرداخت و عبارت‌های تبلیغاتی توسط وکیل و مشاور مالیاتی بررسی شود.
